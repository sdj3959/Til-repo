## 0519 웹 마스터 실전 (3/N): 워드프레스 콘텐츠와 확장 기능

---

### ✅ 1. 콘텐츠의 두 종류: 글(Post) vs 페이지(Page)

*   워드프레스에서 콘텐츠를 작성하는 방식은 크게 '글'과 '페이지'로 나뉩니다. 이 둘의 차이점을 이해하는 것은 웹사이트의 구조를 올바르게 설계하는 데 매우 중요합니다.

| 구분 | 글 (Post) | 페이지 (Page) |
| :--- | :--- | :--- |
| **개념/목적** | • **시간의 흐름**에 따라 기록되는 콘텐츠.<br>• 블로그 기사, 뉴스, 공지사항 등. | • **시간과 관계없는** 정적인 콘텐츠.<br>• 회사 소개, 연락처, 서비스 안내 등. |
| **시간 순서** | • **역순으로 정렬**되어 최신 글이 가장 먼저 보임. | • 시간 순서의 개념이 없음. |
| **분류 체계** | • **카테고리(Category)**와 **태그(Tag)**를 사용하여 분류 가능. | • 카테고리와 태그를 사용할 수 없음. |
| **URL 구조** | • 보통 날짜나 카테고리가 URL에 포함됨.<br>• e.g., `/2026/05/19/sample-post/` | • 단순하고 계층적인 구조를 가짐.<br>• e.g., `/about-us/` |
| **주요 사용 예시** | 블로그 포스팅, 새로운 소식, 이벤트 공지 | 회사 소개, 이용 약관, 문의하기, 오시는 길 |

---

### ✅ 2. 웹사이트의 옷: 테마 (Theme)

*   **테마**는 워드프레스 웹사이트의 전체적인 **디자인과 레이아웃**을 결정하는 템플릿 묶음입니다.
*   **역할**:
    *   사이트의 색상, 폰트, 헤더/푸터 디자인, 메뉴 스타일 등 시각적인 모든 요소를 제어합니다.
    *   콘텐츠(글, 페이지)가 어떤 모습으로 사용자에게 보여질지를 결정합니다.
*   **커스터마이저 (Customizer)**:
    *   `외모 > 사용자 정의하기` 메뉴를 통해, 코드를 직접 수정하지 않고도 테마가 제공하는 옵션(로고 변경, 색상 선택 등)을 실시간으로 미리 보면서 수정할 수 있습니다.
*   **선택 기준**: 웹사이트의 목적에 맞고, 모바일 환경에서도 잘 보이는 **반응형(Responsive) 디자인**을 지원하며, 사용자 평가가 좋고 **정기적으로 업데이트**되는 테마를 선택하는 것이 중요합니다.

---

### ✅ 3. 웹사이트의 기능 확장: 플러그인 (Plugin)

*   **플러그인**은 워드프레스의 기본 기능에 없는 **새로운 기능을 추가**해주는 "앱" 또는 "확장 프로그램"과 같습니다. (비유: 자동차의 내비게이션, 블랙박스)
*   웹 마스터는 사이트 운영에 필요한 필수적인 플러그인들을 알고 적절히 활용해야 합니다.

#### ➕ 필수적인 플러그인 종류

| 종류 | 역할 |
| :--- | :--- |
| **SEO 플러그인** | • 검색 엔진 최적화(SEO)를 도와주는 도구.<br>• `title` 태그, `meta` 설명 최적화, 사이트맵 생성 등. (e.g., Yoast SEO, Rank Math) |
| **보안 플러그인** | • 악성코드 스캔, 로그인 시도 제한, 방화벽 기능 등을 제공. (e.g., Wordfence) |
| **캐싱 플러그인** | • 페이지를 미리 생성해두어 사이트 로딩 속도를 향상시킴. (e.g., WP Super Cache) |
| **백업 플러그인** | • 웹사이트 파일과 데이터베이스를 정기적으로 자동 백업. **(가장 중요)** (e.g., UpdraftPlus) |
| **콘텐츠 빌더** | • 드래그 앤 드롭 방식으로 코딩 없이 복잡한 페이지 레이아웃을 만들 수 있게 함. (e.g., Elementor) |
| **이커머스 플러그인** | • 웹사이트에 쇼핑몰 기능을 추가. (e.g., WooCommerce) |

*   **주의사항**: 너무 많은 플러그인을 설치하거나, 신뢰할 수 없는 플러그인을 사용하면 사이트 속도가 느려지거나 보안에 취약해질 수 있으므로, 반드시 필요한 플러그인만 엄선하여 사용해야 합니다.

### ✅ 4. 사용자 관리: 역할 (Roles)

*   워드프레스는 여러 사람이 웹사이트를 함께 관리할 수 있도록, 사용자별로 다른 권한을 부여하는 **역할 기반의 사용자 관리** 기능을 내장하고 있습니다.

| 역할 (Role) | 주요 권한 |
| :--- | :--- |
| **관리자 (Administrator)** | • 사이트의 모든 설정과 기능을 제어할 수 있는 **최고 권한**. |
| **편집자 (Editor)** | • 모든 사용자(자신 포함)의 글과 페이지를 작성, 수정, 발행, 삭제할 수 있음. (콘텐츠 총괄) |
| **작성자 (Author)** | • **자신이 작성한 글**만 작성, 수정, 발행, 삭제할 수 있음. |
| **기여자 (Contributor)** | • 글을 작성할 수는 있지만, **발행할 수는 없음**. (편집자나 관리자의 검토 후 발행) |
| **구독자 (Subscriber)** | • 자신의 프로필만 수정할 수 있음. (글 작성 권한 없음) |

---

### 📌 요약

*   워드프레스의 콘텐츠는 시간의 흐름에 따라 기록되는 **글(Post)**과, 시간에 구애받지 않는 정적인 **페이지(Page)**로 나뉩니다.
*   웹사이트의 전체적인 디자인은 **테마(Theme)**를 통해 제어합니다.
*   추가적인 기능은 **플러그인(Plugin)**을 설치하여 확장할 수 있으며, 특히 **보안, 백업, SEO, 캐싱** 플러그인은 필수적입니다.
*   여러 사용자가 협업할 수 있도록, **역할(Role)**에 따라 차등적인 권한을 부여하여 사이트를 안전하게 관리해야 합니다.
